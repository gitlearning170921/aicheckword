# 如何提升审核质量

本文说明如何减少“审核到一半自动结束”、让问题点更具体可改、以及如何通过提示词/技能提升审核质量。

---

## 一、待审文档多时“进行到一半自动结束”

**原因**：批量审核时若单次运行时间过长，可能因超时、断连或页面刷新导致脚本中断，已完成的文件不会丢失，但未审核的文件需要重新上传。

**已做改进**：

1. **单文件失败不影响后续**：某个文件审核失败（如解析错误）会记录到操作日志，并继续审核下一个文件。
2. **整批中断有记录**：若整批审核过程中发生未捕获异常（如超时），会记录一条「审核批次」操作日志，并在 `extra_json` 中标记 `interrupted: true`、`reason`，便于排查。
3. **界面提示**：上传前会提示「文档较多时可能因超时中断，已完成的报告会保存到「历史报告」；建议每批不超过 10 个文件，可分批上传。」
4. **中断后仍可查看已审结果**：中断时已成功审核的报告会写入数据库并可在「历史报告」中查看，剩余文件可分批重新上传再审。

**建议**：每批上传不超过 10 个文件，多批分次审核，避免单次运行时间过长。

### DeepSeek / 批量审核与「并发」说明

- **本工具批量审核多份文档时，是按文件顺序逐份调用 API 的**，不会同时对 12 份文档各发一个请求。
- **单份文档若超过约 3 万字**，会拆成多段审核；在 **DeepSeek** 下这些分段会 **串行**（`max_workers=1`），避免分段并发加重网关压力。
- 若仍出现 **`APIConnectionError` / 连接错误**，多见于 **网络、代理、SSL、防火墙或客户端读超时**，不一定是「并发超限」；官方限流更常见为 **HTTP 429**（本工具已对部分瞬时错误做重试）。
- 侧栏可尝试勾选 **「不使用系统代理」** / **「不校验 SSL」**（证书或代理异常时）；DeepSeek 端已配置 **更长请求超时** 与 **份间轻微间隔**，以降低连续请求下的失败率。

## 二、问题点太粗、不知道改哪里、没有指明文档中的位置

**已做改进**：系统提示词已强化以下要求，模型输出会更具体：

- **location（问题位置）**：必须指明在文档中的**具体位置**（如「第3章 适用范围」「第5页 技术指标」或引用有问题的那句话/段落），不得笼统写「文档中」「全文」。
- **suggestion（修改建议）**：必须给出**可操作**的修改建议（如「将第2段中的“XXX”改为“YYY”」「在 4.1 节补充……」），不得笼统写「请完善」「请补充」。
- 每个审核点只针对**一处**具体问题，避免一条里混入多个不相关问题。

若模型仍不够具体，可结合下文「自定义审核要求」进一步约束。

## 三、审核结果太宽、如何提升审核质量？需要提供什么？

**通用要求 vs 自定义要求**：

- **医疗器械软件通用要求**（如有源二类、注册资料一致性、HTTP/MQTT/蓝牙接口与体系等）已内置为**通用审核点**，无需写在「自定义审核要求」里。请到 **「② 审核点管理 & 训练」** 页的 **「📌 内置通用审核点（有源医疗器械软件二类，含网络接口与体系）」** 中点击「加载为清单」或「加载并直接训练」，将其训练到审核点知识库后，审核时会自动按这些要点执行。
- **「自定义审核要求」** 仅用于**公司或项目特有的补充**（如内部规范、本批审核重点、输出范围约束等），会追加到审核上下文中。

**可提供的信息示例（仅适合放在「自定义审核要求」的补充内容）**：

1. **提示词 / 审核重点**  
   - 例如：「只关注软件说明书与技术要求的一致性，其他维度仅做简要检查。」  
   - 例如：「必须检查文档中的产品名称、型号、版本号是否与技术要求完全一致。」

2. **行业或公司规范**  
   - 例如：「本公司要求所有技术指标必须注明单位与检测方法。」  
   - 例如：「引用标准必须写全称与标准号，不得缩写。」

3. **输出范围约束**  
   - 例如：「仅输出确有问题的事项，不要泛泛而谈；无明显问题时可不输出 info 级别确认。」  
   - 例如：「每个问题必须引用文档中的原文或章节号。」

4. **Skills 式说明**  
   - 若你有一份「审核技能」文档（如检查清单、判定规则），可把关键规则粘贴到「自定义审核要求」中，系统会将其与审核点知识一起交给模型执行。

保存后，上述内容会以 **【自定义审核要求（请严格遵守）】** 的形式追加到审核上下文中，无需改代码即可提升审核质量与一致性。

## 四、相关配置与存储

- **自定义审核要求** 保存在 `app_settings.review_extra_instructions`（MySQL），在「文档审核」页的展开区中编辑并点击「保存自定义审核要求」即可生效。
- 批量审核的「中断」记录在 `operation_logs`，`op_type=review_batch`，`extra_json` 中含 `interrupted`、`reason`、`success_count`、`total_files` 等，可在「操作记录」中查看。
