# 有源医疗器械软件（二类）— 通用审核点与自定义审核要求

## 一、通用要求已内置为「通用审核点」，请训练后使用

**医疗器械软件通用要求**（有源二类、注册资料一致性、软件描述与生命周期、**HTTP/MQTT/蓝牙接口与体系文件**、技术要求与检验、说明书与标签等）**不再放在「自定义审核要求」中**，已改为**内置通用审核点**，由系统在审核时通过审核点知识库参与检索与判断。

**使用方式**：

1. 打开 **「② 审核点管理 & 训练」** 页。
2. 在 **「📌 内置通用审核点（有源医疗器械软件二类，含网络接口与体系）」** 展开区中：
   - 点击 **「📥 加载为清单（可编辑后再训练）」**：将内置审核点保存为一条清单，可在列表中编辑、再训练。
   - 或点击 **「📥 加载并直接训练」**：不保存为清单，直接训练到审核点知识库，审核时即生效。
3. 训练完成后，第三步「文档审核」会自动按这些通用要点执行，**无需**在「自定义审核要求」中再粘贴大段通用内容。

内置审核点结构见 **`config/default_checklist_有源软件二类.json`**，覆盖约 20 条（含接口与体系）。

---

## 二、「自定义审核要求」仅用于公司/项目特有补充

「自定义审核要求」适合写**非通用、仅本司或本批审核需要**的内容，例如：

- 本公司/本项目的额外规范（如内部编号规则、必须引用的 SOP）。
- 本批审核的重点或范围（如「本次只核对说明书与技术要求一致性」）。
- 输出习惯（如「仅输出 high/medium，不输出 info」）。

**可选示例（复制到「文档审核」页 → 自定义审核要求 → 保存）**：

```
本批审核补充要求（在通用审核点基础上严格执行）：
- 仅输出确有问题或不符合项，不要泛泛而谈；无明显问题时可不输出 info 级别确认。
- 每个审核点必须标明在文档中的具体位置（章节/页码或引用原文），修改建议必须可操作（如「将……改为……」）。
- 若为本公司内部规范或 SOP 有特别要求的，请一并核对并在审核点中引用。
```

---

## 三、为何通用要求要做成「训练审核点」而不是「自定义审核要求」？

- **通用要求**：适用于所有有源软件二类注册与体系文档，内容多、结构清晰（一条一条的审核点），适合做成**结构化审核点**训练到知识库，由 RAG 按文档内容检索相关要点，审核更稳、可维护。
- **自定义审核要求**：是一段自由文本，每次审核都会整段追加到上下文，适合**少量、灵活**的补充说明（如本批重点、公司特有规则），不适合塞入整份通用要求，否则提示过长且与「审核点知识库」功能重复。

因此：**通用要求 = 训练为通用审核点；自定义审核要求 = 仅作补充。**
