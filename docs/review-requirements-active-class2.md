# 有源医疗器械二类产品注册与体系 — 自定义审核要求（示例）

以下内容可直接复制到系统「文档审核」页的 **「📋 自定义审核要求 / 提示词」** 中，保存后每次审核将自动带入。

---

## 一、适用范围与审核重点

本要求适用于**有源医疗器械二类产品**的注册资料及体系文件审核，依据《医疗器械监督管理条例》《医疗器械注册与备案管理办法》《有源医疗器械注册审查指导原则》及 GB/YY、IEC 等标准，重点核对：

- 注册申报资料与产品技术要求、说明书、体系文件之间的一致性
- 条款与法规、标准的对应关系及表述完整性
- 可追溯性（型号规格、软件版本、文档版本等）在全文中的统一性

---

## 二、建议粘贴的「自定义审核要求」正文

```
【有源二类产品注册与体系审核要求】

一、产品与文档一致性（必须逐项核对）
- 产品名称、型号规格、注册单元名称、软件/固件版本号在全文（技术要求、说明书、综述资料、检测报告等）中须完全一致，不得出现同物异名或版本不一致。
- 结构组成、适用范围、预期用途的表述须与产品技术要求、说明书一致；若引用标准或分类目录，须写全称及编号。

二、法规与标准引用
- 引用国家标准、行业标准（GB、YY）、国际标准（IEC 60601 等）时须写标准全称、标准号及年代号；不得仅写“符合国标”等笼统表述。
- 有源产品须明确电气安全标准（如 GB 9706.1、YY 0505）及电磁兼容标准（如 YY 0505）的适用条款；若为豁免或差异，须说明理由并指向具体条款。

三、技术要求与说明书
- 产品技术要求中的性能指标须在说明书中以用户可理解的方式体现；说明书中的“技术参数”须与技术要求一一对应，单位、量纲一致。
- 说明书须包含《医疗器械说明书和标签管理规定》要求的必备条款（如产品名称、型号、注册人、生产地址、禁忌症、注意事项、警示等）；不得遗漏或使用“见技术要求”等替代必备内容。

四、体系与可追溯性
- 若文档涉及设计开发（DHF）、风险管理（ISO 14971）、软件生命周期（YY/T 0664），须在文档中能对应到具体条款或文件编号，避免空泛表述。
- 文档中出现的版本号、修订号、生效日期须前后一致；引用其他文件时须注明文件编号或名称，便于追溯。
- **需求—开发—测试—风险可追溯性（建议写入自定义审核要求时强调）**：（1）《软件需求规范》中填写的**风险ID、用户需求ID**与《风险管理/风险分析》中的对应编号及**CS编号**（或危害-控制编号）在**内容**上须一致；（2）《软件可追溯性分析报告》中的矩阵或文字与《软件需求规范》及设计/开发/测试文档中对**同一ID**的描述须一致；（3）须核查需求、开发、测试、风险四类文档之间**同一追溯ID**是否闭环、指向是否一致，禁止断链或有号无文。（4）若公司有**专门针对可追溯性的程序文件**，须在自定义要求或知识库中附上程序名称/摘录，并要求审核：**项目文档中的追溯规则、矩阵必备项、签批与记录要求**是否与该程序一致。

五、输出要求
- 每个问题必须指明在文档中的具体位置（章节号、条款号、页码或引用原文），不得笼统写“文档中”“全文”。
- 修改建议须可操作（如“将 3.2 节中的‘XXX’改为‘YYY’”“在说明书 5.1 节补充禁忌症：……”）。
- 仅输出确有问题或不符合上述要求的事项；无明显问题时可不输出 info 级别确认，避免泛泛而谈。
```

---

## 三、使用说明

1. 打开「文档审核」页，展开 **「📋 自定义审核要求 / 提示词（可选）」**。
2. 将上方 **「二、建议粘贴的『自定义审核要求』正文」** 中从 `【有源二类产品注册与体系审核要求】` 到 `避免泛泛而谈。` 的整段复制到文本框。
3. 点击「保存自定义审核要求」。
4. 后续进行上传文件审核或文本审核时，系统会自动将上述要求追加到审核上下文中，模型会按此收紧审核范围并突出有源二类注册与体系要点。

可根据本公司产品特点或审评习惯，在正文中增删条款（如增加公司命名规范、模板章节顺序等）。
