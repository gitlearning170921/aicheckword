# Streamlit 长耗时统一交互技能

## 适用场景

- 用户反馈“按钮后卡住/看不到进度/失败就中断”
- 页面需要跨页跳转（例如审核页传到初稿页）
- 目标文件名与模板名不一致、基础文件不齐时仍需尽量继续执行

## 实施步骤

1. **补齐进度回调**
   - 后端任务函数增加或复用 `progress_cb(msg, frac)`。
   - 前端页面创建 `st.progress` + 状态文本占位，并把回调传入任务函数。
   - 任务结束后显式推进到 100%。

2. **做兼容降级**
   - 优先 `st.progress(value, text=...)`。
   - 若旧版报 `TypeError`，降级为 `st.progress(value)` + `st.caption(msg)`。

3. **改成“可部分成功”**
   - 对目标文件先做归一化匹配（如去 `^\d+_` 前缀）。
   - 未匹配项不要整体报错；允许：
     - 已匹配项先执行
     - 未匹配项降级为文本生成（非就地）
     - 缺少基础文件时仅对该文件降级，不影响其它文件

4. **跨页跳转安全**
   - 触发按钮不要直接写控件 key（如 `main_function_nav_page`）。
   - 写 `_pending_*`，在 `main()` 里控件初始化前消费。

## 推荐提示文案

- 进行中：`正在执行…（42%）`
- 可继续：`未匹配到基础文件：A、B；将按非就地方式继续生成`
- 失败定位：包含异常摘要 + 兜底建议（重试/上传基础文件/切换模板）

## 完成后验证

- 点击开始后，能持续看到百分比和阶段文本变化
- 部分目标未匹配时，流程仍有输出（不是整批失败）
- 跨页跳转不再出现 `session_state key cannot be modified` 异常
